Basic Terminolagies

Data structures are used to hold data
While algoritms are used to solve the problem using that data

Abstraction is a method of hiding the implementation details ad showing only the functionality.
Abstraction allows us to view the problem and solution from two different persepctives

  • Logical
    • What the system does / ITs functionality
  • Physical perspectives
    • How the system is implemented or its components.

Logical perspective (Driver view)

As a driver, you interact with the card by using its functions, starting the engine, steering, braking, and acceleration to drive form one place to another.
This is the logical perspective focusing on the cars interface for its intended purpose

Physical perspective (MEchanic view)

How the engine work, how transmission shifts etc. What makes the car function.

Data

Is a collection of facts and figures or a set of values in a particular format.

07 - Stikkord samling/DAT200/Data structures

Specialized format for organizing, processing, retrieving and storing data.
Enable users to access and manipulate data easily.

Like a grocery store, you want to find items grouped by category in the store making it easier to find what you are looking for.

Two types of data structures/types

Primitive data type

Known as built in datatype.
Kan only store one type of data

  • Can store integers, floating points, characters, pointers.
  • Supported by a programming language.

Non-primitive

Can store more than one type of data

  • Array, linked list, stack, queue, tree, graph etc.

Often referred to as derived data types.

Linear data strucutres

Elements are stored in a sequential order. These structures have a linear relationship in memory

  • Arrays, stacks, queues and linked lists
Non-linear data structures

Not stored sequentially.

  • Graphs: Where elements are stored in memory with random order locations.
  • Tree: Hierarchical structure

Algoritms

Step by step instruction that describes the flow of a program or process.

An algorithm is a set of well-defined instructions to solve a particular problem
It takes a set of input(s) ad produces te desired output

"Good" Algorithm

  • A good algorithm should produce the correct outputs for any set of legal inputs
  • A good algorithm should execute efficiently with the fewest number of steps as possible
  • A good algorithm shold be designed in such a way that others will be able to understand it and modify it to specify solutions to additional problems.
