Graphs

Definition:
A graph is a data structure consisting of:

  • A set of vertices (nodes).
  • A set of edges that connect pairs of vertices, representing relationships between them.
    Formally:
    G=(V,E)
    Where:
    • V(G): finite, non-empty set of vertices.
    • E(G): set of edges (pairs of vertices).

Types of Graphs

  • Undirected Graph: Edges have no direction.
  • Directed Graph (Digraph): Edges have direction; order of vertices matters.
  • Weighted Graph: Each edge carries a weight or cost.
  • Complete Graph: Every vertex is connected to every other vertex.
    • Edges in complete directed graph: .
    • Edges in complete undirected graph: ​.
  • Trees: Special case of graphs with no cycles.

Graph Terminology

  • Adjacent Nodes: Two nodes connected by an edge.
  • Path: Sequence of vertices connecting two nodes.
  • Connected Graph: There is a path between every pair of vertices.
  • Strongly Connected Component: In a directed graph, every vertex is reachable from every other vertex.

Special Structures

  • Spanning Tree: A subgraph of an undirected connected graph that includes all vertices with n−1n-1n−1 edges and no cycles.
  • Minimum Spanning Tree (MST): A spanning tree with the minimum possible sum of edge weights.
    • Found using Prim's Algorithm and Kruskal's Algorithm.

Graph Representations

  • Adjacency Matrix:
    • Good for dense graphs.
    • Memory: .
    • Fast connectivity checks.
  • Adjacency List:
    • Good for sparse graphs.
    • Memory: .
    • Efficient for finding adjacent vertices.

Graph Algorithms

  • Traversal:
    • Depth-First-Search (DFS): Explore as far as possible along a branch before backtracking.
    • Breadth-First-Search (BFS): Explore all neighbors at the current depth before going deeper.
  • Shortest Path:
    • Dijkstra’s Algorithm for weighted graphs.
    • BFS for unweighted graphs.

Applications

  • Networking (routing).
  • Social networks (connections).
  • Pathfinding (maps, games).
  • Dependency resolution.
